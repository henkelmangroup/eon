Recycling¶

We have developed a method of saddle point recycling that can significantly reduce the computational cost of the AKMC algorithm. [1]

Suppose we are in reactant state \(R_0\), and we have discovered a series of saddles and their corresponding products, \(S_i\) and \(P_i\), respectively. Once we have reached confidence that we have found all energetically relevant processes, we select one of these processes and move to the corresponding product state. For this example, let us assume that we have selected the process with saddle \(S_0\) and product \(P_0\). If we have found \(N\) energetically relevant processes in state \(R_0\), we can make suggestions of the saddle geometries \(G_i\) for saddles leading out of state \(P_0\)

\(G_i = P_0 + (S_i - R_0)\)

We use a min-mode following algorithm to converge these suggested saddle points. To reach confidence again in state \(P_0\), we need only perform saddle searches in the region around the atoms that moved significantly from state \(R_0\) to state \(P_0\), resulting in a significant reduction in computational costs. If this region is local, the overall cost does not increase with the total system size.

[Recycling] Options¶

use_recycling: Turn recycling on and off.

default: True

move_distance: The distance an atom must move to be considered in the “hole”.

default: 0.2

displace_moved_only: When performing saddle search displacements, only use atoms in the vicinty of the hole as displacement epicenters.

default: False

save_suggestions: If true, the suggestions made by saddle recycling are saved to the “saddle_suggestions” directory of the state directory for which the saddles are being suggested.

default: False

active_region: Defines a region around the atoms that moved in the previous KMC step to target new saddle searches in the current state. For dynamics based saddle searches, atoms in the active region will have mass weighting applied; for min-mode following saddle searches, only these atoms will be selected for displacements. active_region represents a multiple of [Structure Comparison] neighbor_distace.

default: 1.0

mass_weight_factor: Atoms that are not in within the active region have their atomic masses multiplied by this factor.

default: 1000.0

References
