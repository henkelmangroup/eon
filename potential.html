Potential¶

potential: the type of potential to execute

default: lj

options:

vasp: Vienna Ab-Initio Simulation Program (VASP) interface. [1]

lammps: The LAMMPS potentials. Detailed documentation can be found

on the LAMMPS Potential page. [2]

ase: Atomic Simulation Environment (ASE) interface. Detailed

can be found on the ASE Interface page. [3]

eam_al: Embedded atom method parameterized for aluminum.

qsc: Quantum Sutton-Chen potential, for FCC metals.

bopfox: Bond order potential, for metals.

emt: Effective medium theory, for metals.

lj: Lennard-Jones potential in reduced unit [4]

morse_pt: Morse potential for platinum.

lenosky_si: Lenosky potential, for silicon. [5]

sw_si: Stillinger-Weber potential, for silicon. [6]

tersoff_si: Tersoff pair potential with angular terms, for silicon. [7]

edip: Environment-Dependent Interatomic Potential, for carbon. [8]

zpice: Water on platinum.

tip4p: Point charge model for water. [9]

mpi: Communicate with a MPI process to calculate energy and forces.

More information can be found on the MPI Potential page.

ext_pot: Writes box size and coordinates to the file ‘from_eon_to_ext’

Make a system call for the binary ‘ext_pot’, which should read the coordinates and return the energy and forces to the file ‘from_ext_to_eon’

log_potential: If true write timing information about each force call to client.log

default: true if potential is mpi, vasp, bop, or bopfox

otherwise false
